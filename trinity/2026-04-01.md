## Idea 3: Shared Context Board 2.0: CRDT-based Sync
- **Concept:** Replace current markdown-based shared context with a CRDT (Conflict-free Replicated Data Type) enabling real-time sync across agents without merge conflicts.
- **Problem Fit:** Markdown files cause race conditions when multiple agents write. CRDT solves consistency.
- **Simplicity:** Requires Yjs or Automerge library. Adds complexity but solves real pain.
- **Maintenance Cost:** Medium-High. CRDT libraries evolve; need to manage schema migrations.
- **Bloat Signals:** High for current needs. Over-engineering if writes are infrequent. New paradigm to learn.
- **Utility Score:** 5/10 (discard)
